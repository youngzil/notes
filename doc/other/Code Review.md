

## Code Review

codereview（CR）一直以来在软件行业被视为提升代码质量的一种有效的方式，也被视为一种工程师文化的代表。

许多团队没有Code Review环节，或者因为追求项目快速上线，认为CR浪费时间；或者团队成员缺少CR观念，认为CR的价值并不大。所以想要推动CR在团队中的实施，最最重要的一点便是增强团队成员对CR环节的认同感。


1. what—什么是CR
2. Why—为什么要引入CR
3. Who—参与CR的主体
4. Where—不同的研发节点进行CR
5. when--什么时候引入CR
6. How--如何进行CR
